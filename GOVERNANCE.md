# Governance

context-still is currently maintained by the repository owner.

## Decision Making

Maintainers prioritize:

1. Local-first operation.
2. Auditability of knowledge, evidence, and automation.
3. Clear separation between candidates, diagnostics, and promoted knowledge.
4. Stable MCP and CLI workflows.
5. Focused changes over broad rewrites.

Substantial changes should start as an issue or design note before implementation.

## Documentation Ownership

- `README.md` and `README.jp.md`: project overview and entry points.
- `spec/pub/`: public user/operator documentation.
- `spec/docs/`: internal design notes and implementation plans.

## Release Status

The project is pre-1.0. APIs and schemas can change, but changes should be documented and migration impact should be explicit.
