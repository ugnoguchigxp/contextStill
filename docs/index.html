RAGとの違い

Typical RAG

  • documents を chunk 化して検索
  • 検索結果を prompt に注入
  • 実行結果が知識品質へ戻りにくい

memoryRouter

  • sources から rule/procedure を蒸留
  • token budget 内で context pack をコンパイル
  • feedback loop で knowledge を進化
